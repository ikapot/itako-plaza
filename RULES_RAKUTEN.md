# 🕉️ Zen-LTC-Quant：絶対的な掟 (RULES_RAKUTEN.md)

このファイルは、本プロジェクトにおける AI（Antigravity）および開発者の行動指針を定義したものである。新しいコードの提案、リファクタリング、デバッグを行う際は、必ずこの「掟」を最優先事項として確認すること。

## 1. 銘柄固定 (LTC/JPY 専用)
- 軍資金 2,000円での運用効率を最大化するため、取引銘柄は **symbolId: 10 (LTC/JPY)** に固定する。
- 他の銘柄（BTC/ETH等）への浮気は、マスターの明示的な許可がない限り一切禁ずる。

## 2. 1秒の壁 (Rate Limit)
- 楽天ウォレット API の最新制限に基づき、すべての API リクエスト間隔は **1,000ミリ秒（1秒）以上** 固定で空けること。
- 非同期（Asyncio）環境下においても、Lock 制御等を用いてこのリミットを厳律に遵守せよ。

## 3. TICKER の真実 (Flat Structure)
- WebSocket の TICKER 配信は、`channel` や `data` といった入れ子構造を持たない「フラットな JSON」である。
- パース時は直接 `bestBid`, `bestAsk`, `last`, `timestamp` を参照せよ。存在しない階層へ深く潜る愚（パースエラー）を繰り返してはならない。

## 4. 型の厳守 (String to Float)
- API/WebSocket から届く価格・数量データは常に `string`（文字列）型である。
- 計算（RSI, EMA, ATR 等）に供する前に、必ず `float()` を用いて数値型へ厳格に変換せよ。

## 5. 環境への配慮 (Pure Text Logging)
- Windows コンソール（CP932）環境での文字化けおよび `UnicodeEncodeError` を防ぐため、ログ出力（`print`, `logger`）から**絵文字および特殊記号を完全に排除**せよ。
- ログは視覚的な華やかさではなく、情報の堅牢性と正確性を最優先とする。
