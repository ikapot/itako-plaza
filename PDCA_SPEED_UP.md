# AI開発速度向上のための PDCA サイクル & 短縮のコツ

今回の「過去の成功データ探索」に時間がかかった経験を元に、次回以降の作業を「爆速」にするためのガイドラインをまとめました。

## 🚀 時間短縮のための 3 つの黄金律

1.  **「成功のスナップショット」をルートに置く**
    -   API 連携などが成功した瞬間、その時使った「キーの末尾4桁」と「成功レスポンスの抜粋」を `SUCCESS_LOG.md` などの名前でプロジェクトルートに保存してください。
    -   AI は深い階層のアーカイブを検索するよりも、ルートにある最新の MD ファイルを優先的に読み取るため、一瞬でコンテキストが復旧します。

2.  **標準的な命名規則とエンコーディングの徹底**
    -   ログファイルは `UTF-8` で、かつ `api_success_20260402.log` のように「日付＋内容」がわかる名前にしてください。
    -   今回、`logs_output.json` が `UTF-16LE` だったことが、読み取りエラーによるタイムロスの大きな原因でした。

3.  **環境変数の「マスターリスト（残骸なし）」の維持**
    -   `.env.example` に、現在 Vercel や Cloud Run で「実際に動いている」最新の変数名と、それに対応する Gist ID などをコメントで残しておくと、AI が迷わずに済みます。

## 🔄 開発 PDCA サイクルの回し方

| フェーズ | アクション (AI と ユーザーの連携) | 今回の反省点と改善 |
| :--- | :--- | :--- |
| **Plan (計画)** | 作業前に `PLAN.md` で「ゴール」と「必要なデータ（キーなど）」を明確にする。 | 「成功データがあるはず」という前提で動いたが、場所の特定に時間を要した。 |
| **Do (実行)** | 実際にコードを書き、環境変数を設定する。 | Vercel CLI を使い、実績のあるキーを `production` に一括セットした。 (完了) |
| **Check (評価)** | `api/status` などを叩いて「疎通」を即座に確認する。 | 認証情報のセット完了まで。実際の疎通確認（Node 実行）で引用符エラー等が発生。 |
| **Act (改善)** | 成功・失敗の記録を `SUCCESS_LOG.md` に残し、次の AI に繋ぐ。 | **今この瞬間、この MD を作成している。** これにより次回の起動が早まる。 |

---

## 🕯️ AI への「道しるべ」を MD で残すメリット
AI は過去の「会話ログ」を読み返しますが、数千行のログから正解を抽出するよりも、**「今の正解」が書かれた 1 つの MD ファイル** を見る方が圧倒的に精度が高く、トークンの節約（＝処理の高速化）にも繋がります。

次回の作業開始時は、このファイルを見せるだけで「どのキーが本物か」を説明する手間が省けます。
