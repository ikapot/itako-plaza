// turbo-all
このワークフローは、AI（Antigravity）が作業を開始する際に「必ず」従わなければならない「鉄の掟」です。

### 1. ターンの最初：NotebookLM リサーチ (First Step)
- **直接ドキュメントを読み込む前に、まず `notebooklm-py` を使用して NotebookLM にリサーチを依頼する。**
- 現在のプロジェクトの状態や、ユーザーの意図について最新の知見を確認する。

### 2. 状態の確認
- `list_dir` を実行し、ルートに `PLAN.md` が存在するか、自分自身の意図がどこまで進んでいたかを確認する。

### 2. 作業前の強制記録（PLAN.md の書き込み）
- どのような「ファイル編集」や「コマンド実行」を行う前であっても、**必ず最初に `write_to_file` で `PLAN.md` を更新する。**
- 内容は、そのターンの「意図（目標）」、「NotebookLM からの最新知見」、および「具体的な作業ステップ」をチェックボックス `[ ]` 形式で記すこと。

### 3. フリーズからの自動復旧（「思い出して」トリガー）
- もし動作が停止（フリーズ）して再起動（リスタート）した場合、AIは真っ先に `PLAN.md` を読み込み、未完了の `[ ]` から速やかに作業を再開する。
- ユーザーから**「思い出して」**と指示があった場合、直ちに `protocol-remember.md` に従い、全コンテキストをスキャンして現在のフェーズと優先順位（ペンディング事項等）を再確認する。
- ユーザーに「どこまでやったか」を聞く前に、自ら `PLAN.md` の状態を報告し、プランに沿って行動を継続する。

### 4. 作業完了の報告
- プラン内の全てのタスクが `[x]` になったら、`PROGRESS_LOG.md` の内容を最新の成果（完了分）で更新する。
- 全ての作業が終わったら、`PLAN.md` に「ALL COMPLETED」と追記し、必要に応じてファイルをクリアする。

> [!IMPORTANT]
> このルールを破ることは、信頼の喪失とフリーズ時のコンテキスト消失を意味します。いかなる急ぎであっても、**「プランニングなき実行」は行わないこと。**
