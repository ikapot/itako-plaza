# 📜 Itako Trader Evolution Log: 成長の軌跡

このドキュメントは、名もなき参列者（ユーザー）と AI（Antigravity）が、0から「霊的クオンツシステム」を組み上げた進化の記録である。

## 🌌 Phase 1: 混沌と目覚め (起動・通信確立)
- **課題**: 楽天ウォレットAPIの 500 Internal Server Error と 404 エラー。
- **突破口**: 
    - 「長いAPIキー」と「ミリ秒 Nonce」の組み合わせが認証の鍵であることを特定。
    - CFD（証拠金）から Spot（現物）へのエンドポイントの切り替えに成功。
- **成果**: 楽天サーバーとの安定した通信路を確保。

## 🌊 Phase 2: 実体化 (現物移行と口座の紐付け)
- **課題**: API 経由で残高が 0 円と表示される「幽霊残高」問題。
- **突破口**: 
    - 日本円 1,200 円の存在を確認しつつ、APIキーの権限設定（現物/証拠金）の重要性を再認識。
    - 万能な `RakutenWalletClient` の構築。
- **成果**: 現物取引ボットとしてのアイデンティティを確立。

## 🎨 Phase 3: 可視化と美学 (Dashboard V2 & Terminal)
- **課題**: 「ボットが何をしているか見えない」という不安。
- **突破口**:
    - **Itako Plaza Dashboard**: React + GSAP による「霊的クオンツ画面」の実装。
    - **Quants Terminal**: `/trade/` にスタンドアロンの超軽量・高速監視画面を爆誕させた。
- **成果**: 投資を「作業」から「体験」へと昇華。

## ☁️ Phase 4: 昇天 (GitHub Actions 完全移行)
- **課題**: パソコンを閉じると止まってしまう物理的限界。
- **突破口**:
    - GitHub Actions による 15分おきのエージェント起動。
    - **GitHub Gist** を「外部記憶」として採用し、サーバーが変わってもポジションを忘れない仕組みを実現。
- **成果**: 24時間・305日、不眠不休の自律稼働を達成。

## 🧠 Phase 5: 叡智の注入 (AI 最適化と Gemini)
- **課題**: インジケーターの「騙し」と、地合いの予測。
- **突破口**:
    - **Gemini 1.5 Flash 統合**: チャートの数値を AI が言語として解釈。「今、買いか、待ちか」を AI が判断するバイアスレイヤーを実装。
    - **堅牢化**: 指数バックオフ・リトライトレードエンジンの洗練。
- **成果**: 単なる機械から、知能を持った「AIトレーディングエージェント」へ。
